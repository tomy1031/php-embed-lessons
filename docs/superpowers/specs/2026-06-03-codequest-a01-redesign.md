# CodeQuest 化 ＆ A01再リデザイン 仕様（ユーザーフィードバック反映）

日付: 2026-06-03 / 対象: 既存 php-wasm 学習システム（`lessons/`、`src/`）

A01リデザイン公開後のユーザー要望を反映。**ストーリー駆動（CodeQuest）**と**初心者UXの徹底**へ。

## 1. 世界観と一貫性（CodeQuest）
- 教材全体を「**CodeQuest**」という物語にする。プレイヤー＝学習者が主人公。
- 物語とプログラミングを**一貫**させる：各レッスン＝「クエスト」。学んだコードで物語が進む。
- **表紙(目次) `lessons/index.html`** を世界観のある“冒険の書／ワールドマップ”に。各クエストに一言の物語フック（例：A01「はじまりの村で名乗りを上げよ」）。Phase A/B/C を「章」に。

## 2. 操作チュートリアル（最初に必ず）
- 学習者は操作が分からない前提。A01の冒頭に**チュートリアル**を置く：
  - 「これは実際に動くよ。まず下の **▶ じっこう（実行）ボタンを押してみよう**」と促す最初の `<php-run>` か簡単な演習。
  - エディタは「ここに書き換えられる」「書き換えたら ▶実行 で結果が出る」と説明。
  - 「↺ もとに戻す」「答えを見る」の使い方も一言。

## 3. コメント `//` を説明してから使う
- スターターコードに `//` を出す前に、本文で「`//` から行末まではコンピュータが無視する**メモ（コメント）**。人が読むための注釈だよ」と説明する。
- 以降のスターターのコメントは、その理解の上で使う。

## 4. いきなり全部書かせない（穴埋め中心）
- 導入は**例文を先に提示**し、学習者は**小さな1か所を書き換える／埋める**だけ。徐々に書く量を増やす。
- 最初の演習は「`echo` の文字を自分の名前に変えるだけ」等、1ステップの変更から。

## 5. 変数の説明を作り直す（「箱」をやめる）
- 「値をしまう箱」はやめる（「箱ってなんだよ」となる）。
- 動機づけ重視：「同じ文字を毎回 `echo` で書くのは大変。**一度『名前』をつけて覚えさせれば、何度でも呼び出して使える**。それが変数。」
- 比喩は「**冒険者の名簿に名前を書いておく**」「プレイヤー情報を覚えておく」など、CodeQuestの文脈で。

## 6. ストーリー仕立ての具体例（A01の核）
順序イメージ（やさしく・例文先行）：
1. **タイトル画面**：`$title = "CodeQuest"; echo $title;` → まず ▶実行（チュートリアル）。
2. **名乗り（変数の導入＝動機づけ）**：
   - まず「毎回書くの大変」を体験 → 変数に名前を入れて再利用：
     ```php
     $name = "ゆうしゃ";
     echo $name . "よ、よくぞ来た！";
     echo $name . "の ぼうけんが はじまる！";
     ```
   - 演習：`$name` を**自分の名前に書き換える**だけ（例文あり）。物語が自分の名前で進む。
3. **数とステータス**：`$hp` などを設定して表示（CodeQuestのステータス）。
4. **つなぐ `.`** → 仕上げ。
5. **配列への布石（最後に予告）**：一人分は変数でOKだが情報が増えると大変 → 連想配列でまとめると整理できる、を“次回予告”として見せる：
   ```php
   // 情報が増えるとバラバラで大変…
   $name = "アレン"; $lv = 1; $hp = 20; $mp = 5;
   // 次回：こうまとめられる！
   $yuusha = ["name"=>"アレン", "lv"=>1, "hp"=>20, "mp"=>5];
   ```
- 各ステップが**物語を前進させるタスク**になるように（名乗り→旅立ち→…）。

## 7. UX修正（プラットフォーム全体・全レッスンに効く）
- **エディタが暗すぎる**：演習エディタ（CodeMirror）を**明るく読みやすく**する。羊皮紙系の明るい背景＋濃い文字、または高コントラストのテーマに。`body.rpg php-exercise .editor-host` 配下で確実に可読に。プレースホルダ/カーソル/シンタックス色のコントラストAA以上。
- **答えを最初から見せない**：`模範解答（答えを見る）` ボタンは**最初は出さない**。学習者が **一度 ▶実行 した後にだけ表示**する（または「ヒント→答え」の二段）。答えのコードが初期状態で見えないようにする。
- `php-run`（表示例）の出力は従来どおり見せてよい（これは“例”なので答えではない）。

## 8. 実装メモ
- `src/components/php-exercise.ts`：solutionボタンを実行後に表示するロジック（初回 `runCode()` 後に `.solution` を表示）。`solution` 未指定なら従来どおり非表示。
- `src/editor/codemirror.ts` / `src/styles/course.css`：エディタ可読性（明るいテーマ）。
- `lessons/phase-a/a01-variables.html`：本仕様6の流れへ全面書き換え（チュートリアル＋例文先行＋穴埋め＋コメント説明＋変数の動機づけ＋配列予告）。採点は従来どおり expected 全文一致、solution は実行後表示。
- `lessons/index.html`：世界観のあるクエスト目次。
- 検証：単体／tsc／build／course-integrity E2E（solution→正解）を緑に。E2Eで solution ボタンが実行後に出る点に注意（テストは「実行→solutionクリック→実行→正解」へ調整可）。
- A02/A03 も後で同じ CodeQuest 文脈・UX へ揃える。
