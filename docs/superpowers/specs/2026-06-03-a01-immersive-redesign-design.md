# A01 没入リデザイン 設計書（会話・ポイント・図解・UI統一）

日付: 2026-06-03 / 対象: php-embed-lessons（Vite + Web Components, `body.rpg` テーマ）
前提: 既存の CodeQuest A01（`lessons/phase-a/a01-variables.html`）と `php-run`/`php-exercise` を改修。

## 背景・ねらい
初心者ユーザーの実地フィードバックを反映し、A01を「文字中心」から「**図解・イラスト・キャラクター会話中心**」の没入型へ刷新する。あわせて演習UIの迷い（書く場所・空の白エリア・実行ボタン位置）を解消する。

## 要件（ユーザー指示）
1. 「次のクエスト予告（配列）」セクションを**削除**。
2. 演習エディタに**書く余白**を確保（どこに書くか分かる）。
3. 最初は**フル記述でなく「値を変えると表示が変わる」体験**から。
4. **空の白い出力エリアが不要**（実行前は出さない）。
5. **不自然な半角スペースを全廃**し、**普通の日本語（漢字：冒険・始まる・実行 等）**に。
6. 「ポイントは3つ」等のための **point表示コンポーネント**を新設。
7. **文字より図解・イラスト重視**＋**キャラクター会話コンポーネント**（吹き出し会話）で没入＆楽しく。今回まとめて絵を生成。
8. **実行ボタンの位置を全コンポーネントで統一**（コードのすぐ下・左）。

## 新コンポーネント

### 1. `<char-talk>` キャラクター会話
- 記法: `<char-talk speaker="mira" side="left">セリフ本文</char-talk>`
  - `speaker`: キャラID（`mira`=賢者ミラ / `allen`=見習い勇者アレン / `slime`=スライム / 以降の仲間も追加可能）。
  - `side`: `left`(既定) / `right`。アイコンの左右配置。
  - 本文は textContent（HTMLは最小限、`<b>` `<code>` 程度を許可）。
- 描画: `<div class="char-talk side-X"><img class="avatar"><div class="bubble"><span class="name">名前</span><p>…</p></div></div>`
  - アイコンは**円形**（CSS `border-radius:50%`＋`object-fit:cover`）。GPT生成のキャラアイコン画像を使用。
  - `speaker` → {名前, アイコンパス, アクセント色} のマップを**コンポーネント内テーブル**で定義（仲間追加はこのテーブルに1行足すだけ）。
  - 画像未生成でも崩れないプレースホルダ（背景色＋頭文字）を持つ。
- 役割: 「地の文」の説明を先生役（ミラ）＋生徒役（アレン）の会話に置換し、初心者の疑問を代弁する。

### 2. `<point-box>` ポイント表示
- 記法: `<point-box title="ここがポイント"><li>…</li><li>…</li></point-box>`（`title` 省略時は「ここがポイント」）。
- 描画: 💡アイコン＋タイトル＋金枠カード。子の `li` を番号チップ付きリストに整形。
- 「ポイントは3つ：」＋ `ul` を置換し、視覚的にスッキリさせる。

両コンポーネントとも `src/components/` に Custom Element として実装し、`src/index.ts` で `customElements.define` 登録。単体テスト（jsdom）を追加。

## プラットフォーム改修（php-run / php-exercise）

### 実行ボタン位置の統一
- `php-run`/`php-exercise` とも**縦積み**に統一：`コード → 操作バー（▶実行を左端）→ 結果`。
- 横並び(split)レイアウトは**廃止**（`decideLayout` の split 分岐は使わない／compact は1行表示の簡略のみ）。`php-run.layout-split` のグリッドCSSは撤去。
- `php-exercise` も同様に縦積み（`.panes` の2カラムグリッドを廃し、エディタ上・結果下）。

### 空の白エリア撤去・結果は実行後のみ
- `php-run`/`php-exercise` の `.output` は**実行前は非表示**（DOMに空のクリーム箱を出さない）。最初の `▶実行` で表示する。
- `php-run` の実行前ガイドはボタン付近の小テキストに留め、空の大きな出力枠は出さない。

### エディタの書く余白
- 演習エディタ（CodeMirror host）に**最低6〜8行ぶんの min-height**を与え、どこに書くか明確化。プレースホルダ/カーソルは既存の明るいテーマを維持。

### 既存仕様の維持
- 採点（expected 全文一致、正規化）、`solution` は初回実行後に表示、localStorage 保存（starterハッシュ込みキー）は**そのまま維持**。
- E2E セレクタ（`button.run` / `.result` / `.output` / `.cm-content`）を壊さない。

## キャスト（GPTでアイコン生成）
- **賢者ミラ**: 案内人・先生役。やさしい。アクセント色＝魔法紫。
- **アレン**: 見習い勇者＝学習者の分身。素朴な疑問を言う。アクセント色＝青。
- **スライム（ぷに）**: 合いの手・マスコット。必要に応じて登場。緑。
- 将来の仲間: `speaker` テーブルに追記で増やせる。

## 画像（今回まとめて生成）
- キャラ円形アイコン: ミラ / アレン / スライム（正方形・顔中心・背景シンプル＝円形くり抜き前提）。
- シーン挿絵: 必要箇所に2〜4枚（タイトル/村、変数=名札、レベルアップ、ステータス）。既存の良い3枚は流用可。
- 生成は ChatGPT（Claude-in-Chrome 経由）。**各DLごとに md5 で重複チェック**し、Viteの同一ハッシュ統合による“同じ絵”事故を防ぐ。
- 配置: `lessons/assets/img/a01/`（キャラは `char/` サブ）。

## A01 構成（改訂後の流れ）
1. **タイトル/つかいかた**: ミラが「▶実行を押してみよう」と案内（会話）。最初の実行ボタンに説明。
2. **echo**: 画面に文字を出す。会話＋point-box（`" "` で囲む・`;` で終わる・`//` はメモ）。
3. **変数の動機**: 「同じ言葉を何度も書くの大変」をアレンが言う→ミラが変数を提案。
   - **演習①: 値だけ変える**（完成コードの `$name`/数を変えて出力が変わるのを体感）。
4. **変数の便利さ**: point-box（①繰り返し使える ②あとで変えられる＝レベルアップ）。レベルアップ実演。
5. **数・連結**: 文字は囲む/数は囲まない、`.` でつなぐ。`" "` の中に記号も入る地ならし。
6. 段階演習（穴埋め→記述）→ **仕上げクエスト**（半角スペース・半角かっこの注意つき）。
7. **実アプリも同じ**（②）。**配列予告は削除**。
