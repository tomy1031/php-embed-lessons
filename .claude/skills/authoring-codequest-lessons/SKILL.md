# CodeQuest レッスン制作（初心者向け教材）

A01（変数）の制作で確立した品質基準。**完全初心者・非エンジニア・数字が苦手・ゲーム好き**を想定し、これと同等のケアを全ページで維持する。新ページ（A02/A03…）は必ず本スキルに沿う。

対象: `php-embed-lessons`（Vite + Web Components, `body.rpg` テーマ）。レッスンは `lessons/phase-*/aNN-*.html`。

## 0. 進め方（毎回）
1. **ブレスト→仕様→計画**（superpowers: brainstorming → writing-plans）。新コンポーネントや大きな構成変更は設計を先に。
2. **作成と review はエージェントを分ける**。本文を書いたら、別エージェントに**初心者ペルソナ**で辛口レビューさせ、所見を反映。最低1ペルソナは「**文系・数学が大の苦手・ゲーム未経験のさき(32)**」を入れる（記号・式・全角半角で詰まる視点）。指摘→修正を1〜2巡。
3. **検証ゲート（必須・全部緑にしてから公開）**: `npm run test`(vitest) / `npx tsc --noEmit` / `npm run build` / `npx playwright test`（特に course-integrity = 各採点演習の `solution` が `expected` を出すこと）/ Preview か `preview_inspect` で実機確認。
4. **公開**: commit → `git push origin master` → GitHub Pages（`gh run watch <id> --exit-status`）→ ライブで HTTP 200＋新文言を `curl|grep` 確認。公開は公開操作なので**ユーザー承認を取る**（セッションごと）。

## 1. 文章・言葉づかい（最重要）
- **普通の日本語・漢字**で書く（冒険・始まる・実行）。**不自然な分かち書き（語間の半角スペース）禁止**。出力文字列も自然に（「の冒険が始まる」。余分なスペースを入れない）。
- **未説明の専門語・メタ語を出さない**。「地の文」「スターター」「string/int/型」などは使わない。「コード」は初出で**「コード（プログラム）」**と添える。
- **記号は使う前に説明**: `$ . ; " // \n ( ) =>`。読みも添える（`;`＝セミコロン＝文末の「。」、忘れると動かない）。
- **概念は"数学に見えない言葉"で**:
  - `=` … 「**右にあるものを、左の名前に入れる**（読むは右→左）」。算数のイコールではない。
  - `.` … 「**言葉をのりでくっつける**」。`+`でも小数点でも句点でもない。
  - 変数 … 「**名前をつけて覚えさせる**」「**あとから中身を入れ替えられる**（レベルアップ）」。「箱」だけの比喩は避け、図で示す。
- **動機づけを先に**（なぜ変数？＝毎回打つのが大変／あとで変えたい）。
- **現実への接続**を入れる（買い物アプリ等）。題材は**実在の商品＋円**（「やくそう」等のゲーム内アイテムや「ゴールド」だけにしない）。
- 不要な先取り（次回予告など）は**入れない**。ユーザーが不要と言ったら削る。

## 2. 構成・教え方
- **物語＋会話で進める**（地の文の説明を `<char-talk>` の会話に置き換える）。文字より図解・イラスト重視。
- **順序は「コード→▶実行→解説」**。`point-box`（要点）や解説は**コードの後**に置き、動かして確認してから理解させる。
- **操作説明は最初のエディタのすぐ下**に置く（書き換え／▶実行／↺最初に戻す／答えを見る）。
- **演習の難易度カーブ**: ①値を1つ変える→表示が変わる体験 → ②穴埋め（記号・空白は配置済みで変数1つだけ直す）→ ③徐々に記述 → ④**ボス＝全部の総合**。
- **目に見えない文字を打たせない**: 半角スペース・半角かっこ等は**最初からコードに入れておく**か、checker で吸収（§4）。学習者には「見える」要素だけ操作させる。
- **記入スペースを確保**: 自分で改行させない。starter に空行を入れる。`dedent`（src/components/extract.ts）は**先頭・末尾の空行を除去**するので、空行は**下にコメントのアンカー**を置いて中間に保持する。

## 3. コンポーネント（この資産を使う）
- `<char-talk speaker="mira|allen|slime" side="left|right" avatar="../assets/img/aNN/char/x.png">セリフ</char-talk>` … 丸アイコン＋吹き出し。話者表は `src/components/char-talk.ts`（**仲間追加は1行足すだけ**）。画像失敗時は頭文字プレースホルダに退避。
- `<point-box title="ここがポイント"><li>…</li></point-box>` … 要点カード（「ポイントは3つ」を置換）。番号付きで出るので、**出力例の提示には使わない**（番号が混乱の元）。出力例は callout＋`<code>` で。
- `<php-run>` … 表示＋実行。**自動実行しない／▶実行ボタン必須／出力は実行後のみ表示**。最初の1つに操作ヒント。
- `<php-exercise expected="…" solution="…" [boss]>` … 採点演習。上部に**種別バッジ自動表示**（🗡クエスト=expectedあり／✏️じゆうれんしゅう=expectedなし／👾ボス戦=`boss`属性）。solution は初回▶実行後に出る。出力は実行後表示。
- **各クエストは `<div class="quest">`（金の破線枠）で囲む**＝練習問題のまとまりを明確化。
- **最終問題はボス**: 直前に `<div class="boss-banner">👾　ボスが あらわれた！　👾</div>` ＋ `boss` 属性。ボスは**その章で習った全要素を使う総合課題**にする。
- 実行ボタン位置は**コード直下・左に統一**（横並びにしない・縦積み）。実行前に空の出力枠を出さない。

## 4. 採点（checker）
`src/checker/checker.ts` の `normalize` は CRLF→LF、**全角スペース→半角**、各行末空白除去、全体trim。
- **目視で区別できない違い（全角/半角スペース）は吸収する**方針。新たな"見えない罠"が出たら checker 側で寛容にするか、本文で明示。
- `expected` に**実改行**を入れてよい（HTML属性に改行を直接書く）。`\n` を使う演習はこれで2行出力を採点できる。
- **course-integrity E2E**: すべての採点演習で `solution` が `expected` を生むこと（壊れたら直す）。

## 5. 画像（GPT生成パイプライン）
ChatGPT（Claude-in-Chrome MCP）で生成。**同じ絵柄・同じ世界観**を指定して統一感を出す。
- **キャラアイコン**: 正方形・顔/上半身中央・背景は淡い無地（円形くり抜き前提）。文字・ロゴなし。`lessons/assets/img/aNN/char/`。
- **概念図**: 16:9・淡い背景・絵本風。**正確なラベル（`$name` `$level` 「勇者」等）が要る図は、まず無地で生成→ChatGPTの「編集」機能でラベルを追記→`zoom` で文字を必ず目視確認**（一発生成より確実）。数字（1, 5, 120円）は単体なら概ね描ける。
- **配置と圧縮**: ビューアのダウンロード→`~/Downloads`→`cp` で配置→`sips -Z 320`（アイコン）/`640`（シーン）に縮小。
- **md5 重複チェック必須**: 生成ごとに `md5 -q` で**全画像が別ハッシュ**か確認。Viteは同一バイトのファイルを1つに統合するため、同一だと「同じ絵が複数」事故になる（A01で発生）。
- **参照経路**: `<img src="../…">` は Vite が自動ハッシュ化。`<char-talk avatar>` のような**カスタム属性参照は Vite を通らない**ので `vite.config.ts` の copy プラグイン（`char/` を `dist/lessons/assets/` へコピー）で配る。

## 6. 公開後チェック
- ライブ URL で対象ページ 200、追加文言・画像（ハッシュ名）の 200 を確認。
- 反映が古い場合の「Shift+再読み込み」を案内。
- 画像5点ルール等の事故防止は md5 で再確認。
